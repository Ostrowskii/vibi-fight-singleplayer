Prompt inicial:
crie um diretorio chamado docs e coloca ele no git.ignore

Objetivo:
Padronizar e acelerar a criação dos próximos projetos a partir do Nautilus, evitando ter que reexplicar 
instruções recorrentes como a criação do diretório `docs` e sua inclusão no `.gitignore`.

Organização de arquivos:
Arquivos `.txt` não devem ficar soltos na raiz do repositório. Sempre criar ou mover esses
arquivos para dentro de `docs/`.

Deploy padrão:
O projeto utiliza GitHub Pages, e eu sempre configuro a publicação para pegar os arquivos
a partir do root.

Preferência para app web:
Em aplicações web, evitar 100% barra de rolagem na página inteira. Se houver overflow,
o scroll deve ficar apenas dentro do componente que realmente precisa dele.

Preferência operacional de git:
Quando uma mudança estiver pronta, a IA pode criar commit e fazer push direto, sem pedir
confirmação adicional. Isso deve ser tratado como preferência padrão do Nautilus.

Padrão para projetos online multiplayer:
Se o projeto for multiplayer online, utilizar a lib `vibi-network` (`vibinet`) como base
de rede.
Esse tipo de projeto precisa obrigatoriamente de `usuario` e `sala`, porque sem essas duas
informações a sincronização e a organização da partida ficam ruins.
Cada sala deve representar exatamente uma única partida. Nunca reutilizar a mesma sala para
mais de uma partida, rodada, sessão ou qualquer outra variação. Se quiser jogar de novo,
criar outra sala.
